## Critical Paths (E2E — must work)
- `harnessa run benchmark-1 --mode trio` → full pipeline completes, telemetry written
- `harnessa run benchmark-1 --mode solo` → solo completes, post-hoc eval, telemetry written
- `harnessa run benchmark-1 --mode trio` with eval failure → retry fires, feedback carries, eventually passes or hits max iterations
- `harnessa replay <run-id>` → re-evaluates saved artifacts with correct scores
- Cross-model eval: two different model evaluators produce independent scores, reconciler merges

## Key Interactions to Verify
- Generator subprocess CANNOT read `_eval/` directory (Goodhart boundary)
- Evaluator subprocess CAN read `_eval/` directory (injection works)
- Atomic file handoff: writer creates `.tmp` + renames + writes `.done`; reader waits for `.done`
- `.status` file protocol: orchestrator detects agent errors without polling output files
- Resource limits: subprocess respects memory/CPU limits via preexec_fn
- Port allocation: concurrent benchmarks don't collide on ports
- Graceful shutdown on Ctrl+C: subprocesses killed, partial telemetry saved

## Edge Cases
- API key missing → clear error before any subprocess spawns
- LiteLLM returns max_tokens truncated response → ResponseAdapter detects, triggers retry
- Evaluator rubber-stamps (all scores ≥9) → detected and flagged in telemetry
- Evaluator refuses to be negative → detected, re-prompted or model switched
- Both evaluators fail → run marked FAILED with partial telemetry
- Benchmark too easy (solo scores 9+) → difficulty analysis flags it
- Benchmark too hard (both modes fail) → difficulty analysis flags it
- Generator produces unparseable output → one re-prompt, then FAILED
- Run interrupted mid-generation → cleanup fires, no orphan processes

## Criteria System Tests
- Valid YAML loads and parses correctly
- Invalid YAML (missing fields, bad types) → clear validation error
- Few-shot examples accessible to evaluator prompt
- Weight/threshold values enforced (HIGH/MEDIUM/LOW, 1-10)

## Telemetry Validation
- Run manifest JSON conforms to Pydantic schema
- Model identity captured per-agent (provider + model_id)
- Tool usage logged per-agent via wrapper
- Cost calculation correct (tokens × per-token price)
- Quality trend tracks score evolution across iterations
- Difficulty scoring produces calibration recommendations
